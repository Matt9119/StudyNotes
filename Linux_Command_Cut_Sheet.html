Important Account & Config File Paths
📂 Account Administration Files
File Path Format / Key Fields Description
/etc/passwd user:x:UID:GID:comment:home:shell User account info. Readable by all. The x means password is in shadow.
/etc/shadow user:hash:lastchg:min:max:warn:inactive:expire Hashed passwords + aging policy. Root-readable only. Use lastb / chage to inspect.
/etc/group groupname:x:GID:members Group membership. Add users with usermod -aG or gpasswd -a.
/etc/gshadow group:pw:admins:members Secure group account info. Defines group admins. Root-only.
/etc/sudoers user HOST=(runas) cmds Sudo permissions. Always edit with visudo to prevent syntax errors. Use %group to assign sudo to a group.
/etc/sudoers.d/ Drop-in config files Place additional sudoers rules here instead of editing the main file.
/etc/shells One shell path per line Lists valid login shells. chsh only accepts shells listed here.
/etc/security/ Multiple config files PAM security restrictions — login limits, time-based access, etc.
/etc/hosts IP hostname alias Local DNS resolution. Checked before DNS queries.
/etc/resolv.conf nameserver IP DNS server addresses the system queries.
/etc/hostname Plain text hostname Sets the machine hostname. Apply with hostnamectl set-hostname.
/home/user/.ssh/authorized_keys One public key per line SSH public keys allowed to log in as this user. Must be chmod 600.
/etc/cron.allow One username per line Whitelist of users allowed to use cron.
/etc/cron.deny One username per line Blacklist of users blocked from cron.
/var/run/utmp Binary — use w or who Currently logged-on users.
/var/log/wtmp Binary — use last Successful login/logout history.
/var/log/btmp Binary — use lastb All failed (bad) login attempts.
/var/log/lastlog Binary — use lastlog Most recent login timestamp per user.

📦SCP — Secure Copy

scp -rCopy entire directoryscp -r /data/ alice@host:~/

💡
Key-based auth flow:
1. Generate keys: ssh-keygen -t rsa -b 4096 -f ~/.ssh/mykey
2. Protect private key: chmod 600 ~/.ssh/mykey
3. Push public key to target: ssh-copy-id -i ~/.ssh/mykey.pub alice@host — or manually append to ~/.ssh/authorized_keys
4. Connect: ssh -i ~/.ssh/mykey alice@host
⚠
Private key file must be chmod 600 or SSH will refuse to use it. Public key on the remote server goes in ~/.ssh/authorized_keys and should be chmod 600 as well.

Bash Scripting Basics
Creating & Running a Bash Script — Step by Step myscript.sh
1
Create the file
Use any text editor. Like nano or vim.
vim myscript.sh
2
Add the shebang line — first line of every script
Tells the OS which interpreter to use. Must be the very first line.
#!/bin/bash # This is a comment — describe what your script does # Store a value in a variable (no spaces around =) NAME="World" # Print to the screen echo "Hello, $NAME!" # Run a command and store its output TODAY=$(date) echo "Today is: $TODAY" # Simple if statement if [ -f /etc/passwd ]; then echo "passwd file exists" fi
3
Save and give the file execute permissions
Without execute permission the script won't run.
chmod +x myscript.sh # Verify permissions — should show -rwxr-xr-x or similar ls -l myscript.sh
4
Run the script
Use ./ to run from current directory, or full path. Or call the interpreter directly.
# Run with ./ (must be executable) ./myscript.sh # Or call the interpreter directly (no chmod needed) bash myscript.sh # Run from full path /home/alice/myscript.sh
5
Debug and test
Use -x flag to trace every command as it executes — great for troubleshooting.
# Run in debug/trace mode bash -x myscript.sh # Check syntax without running bash -n myscript.sh # Check the exit code of the last command (0 = success) echo $?
6
Make it available system-wide (optional)
Move or copy to a directory in $PATH so you can call it from anywhere.
# Copy to a PATH directory sudo cp myscript.sh /usr/local/bin/myscript # Now run from anywhere without ./ myscript
