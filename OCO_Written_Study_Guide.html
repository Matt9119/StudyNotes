Cyber Ops in the DoD
Title 50 vs. Title 10
Title 50 — War & National Defense / Intelligence Community Activities
Cyber Exploitation falls here. Military intelligence activity that supports OPE/positioning for future ops. Can happen in Gray or Red space. Covert in nature. Does NOT create cyberspace effects.
Title 10 — Armed Forces / Military Operations
Cyber Attack falls here. Actions that create noticeable D4M effects (in cyberspace or physical domains). A form of fires — typically overt, since it's meant to be noticed. Red space.
Key takeaway

"DoD and IC Interagency Support" and "Covert Actions" can fall under both titles.

D4M
Effects Terminology
Deny
Prevent access to, operation of, or availability of a target.
Degrade
Deny to a level represented as a percentage of capacity.
Disrupt
Deny completely, but temporarily, for a specified amount of time.
Destroy
Deny completely and irreparably.
Manipulate
Control or change information/computers/systems/networks to create physical denial effects.
Hacker Methodology — Tradecraft
Why Tradecraft Matters
Tradecraft is skill acquired through experience — being professional and deliberate. It matters because of:
  • International/political implications of cyberspace ops
  • Collateral effects
  • Intel gain/loss
  • Technical gain/loss
  • Jail
How effective operations employ tradecraft:
  • Accountability
  • Plan
  • Conduct sandboxing / ROC drills
  • Understand the target environment
  • Minimize errors
  • Leave the target as you found it
Redirectors
Used for the following reasons:
  • To get around an obstacle
  • Maneuvering from one host to another
  • Obfuscating the source of an attack
  • No direct route to the target
Hacker Methodology — 6 Phases
The Full Cycle
1
Reconnaissance
Collecting info about the target — usually no direct interaction, often just public tools like Google. The single most important phase.
2
Enumeration / Scanning
Start interacting with the target to find vulnerabilities and determine overall attack surface — i.e. WHAT the target might be vulnerable to.
3
Gaining Access (Exploitation)
The "glamorous" part — but only as good as the recon/enumeration before it. Primary tool: Metasploit.
4
Privilege Escalation (Exploitation)
Target account depends on OS — Administrator/System (Windows) or root (Linux). Includes cracking passwords, post-exploitation modules, local exploits/scripts, and installing backdoors.
5
Covering Tracks
Erase digital signs left during earlier stages — restoring modified files, deleting tools/files placed on target, closing opened ports. Think about this the whole way through, not just at the end.
6
Reporting
The "paperwork" phase — document findings, criticality, how found, and remediation. Includes the Mission Summary and Mission Report (MISREP).
Don't skip ahead

If you don't enumerate thoroughly you may miss an opportunity — and if you didn't look hard enough at the target, the exploit you chose may fail entirely. Access is only as good as the work that came before it.

Tunneling, Proxies & Port Redirection
Why Tunnel
Tunneling forwards traffic through a redirection. Reasons to tunnel:
  • Getting around obstacles like firewalls
  • Obfuscating the source of the operator's attack platform
  • No direct route to the target

SSH Tunnels

Basic SSH login
ssh <username>@<remote_IP> ssh <username>@<remote_IP> -i ~/<identity_key>
Local Port Forward
Listens on your local machine and forwards traffic to a specific remote destination.
ssh -i ~/<identity_key> -L <local_bind_port>:<target_IP>:<target_port> <username>@<ssh_server_IP>
Remote Port Forward
Listens on the remote server and forwards traffic back to a specific local destination.
ssh -i ~/<identity_key> -R <ssh_server_IP>:<ssh_server_bind_port>:<attack_platform_IP>:<listening_port> <username>@<ssh_server_IP>
Dynamic Port Forward
Listens on your local machine and creates a SOCKS proxy to route traffic to any destination reached via the remote server.
ssh -i ~/<identity_key> -D <local_proxy_port> <username>@<ssh_server_IP>
Combining Tunnels
Combine Dynamic + Remote port forwarding for full target interaction.
ssh -i ~/<identity_key> -D <local_proxy_port> -R <ssh_server_IP>:<ssh_server_bind_port>:<attack_platform_IP>:<listening_port> <username>@<ssh_server_IP>
Color key (as taught)

Red = our attack platform / host machine  ·  Blue = SSH server / redirector  ·  Green = target machine

Proxychains

What it's for
Forces connections through a proxy — used to make use of an SSH dynamic port forward or any other local proxy.
SOCKS5 vs SOCKS4
SOCKS5 allows UDP/auth. SOCKS4 does not allow UDP traffic.
Config file
/etc/proxychains4.conf
Rule of thumb

You should always be using SOCKS5.

Netsh Port Relays

What it is
A Windows command-line scripting utility — can display and modify network configs, locally or remotely. Included since Windows 2000.
Use here
Can do tunneling via port forwarding — functionally the same as ssh -L.
Reconnaissance

  • DNS records — nslookup, dig, host, etc.
  • DNS zone transfers
  • Public webpages, blogs, social media, etc.

Metasploit Framework

  • Free, open-source exploitation framework — standardized environment for deploying/managing exploits
  • Runs on Windows, Linux, and macOS
  • Start from Kali with msfconsole
Module variables
  • RHOST & RPORT
  • LHOST & LPORT
  • Payload
  • Target
  • Proxies*
  • ReverseListenerBindAddress*
  • ReverseListenerBindPort*
  • ReverseAllowProxy*

* Advanced settings — configure with setg or set.

Worked example — SSH tunnel + matching msf variables
ssh -i ~/.ssh/redirector_key -D 9050 -R *:8080:77.16.45.130:8080 na@64.12.72.94 setg LHOST 64.12.72.94 setg LPORT 8080 setg Proxies socks5:127.0.0.1:9050 setg ReverseListenerBindAddress 77.16.45.130 setg ReverseListenerBindPort 8080 setg ReverseAllowProxy true

Post-Exploitation Methodologies

Migration & User Tokens

Migration serves a few purposes:
  • Hide what process you're in, to avoid detection
  • Stability
  • Change user privileges

Note: you can go down in privilege level when migrating, but not up.

Good meterpreter TTP

1) Get SYSTEM if you don't already have it (getsystem) → 2) Migrate to a stable SYSTEM-level process → 3) Kill the application that was exploited → 4) Restart the executable in a clean state. This lets you re-exploit the target if your session dies, and restores the service's functionality — reducing suspicion.

Common Web Exploitation Methods
Directory Traversal
Access files/directories outside the web root by manipulating a path parameter.
Command Injection
Application passes unsafe user-supplied data to the system shell — a result of OS task handling and, directly, of not validating user input.
SQL Injection
Data entered from an untrusted source gets used to dynamically construct a SQL query — can modify, delete, or add to SQL tables.
Cross Site Scripting (XSS)
Scripts execute within a user's browser under the security context of the site they're visiting — the browser believes the script is trusted. Can steal victim cookies (session hijack on an authenticated site) or redirect to a malicious domain.

Three categories: Reflected (via email/chat/redirection) · Stored (uploaded/embedded) · DOM-Based (client-side scripts write user data to the DOM)
