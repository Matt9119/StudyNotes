Question 10
Describe the correct order of the Linux boot process, naming all 4 stages and what happens at each one.
Answer
1. Firmware (BIOS/UEFI) — Runs POST, discovers hardware, loads the bootloader into memory.
2. Boot Loader (GRUB) — Loads the kernel into working memory. May allow user to choose which kernel.
3. Kernel — Starts /sbin/init and loads processes needed for the OS to run.
4. User Space — Init system (SysV/Upstart/Systemd) starts services; user can now interact with the host.
Question 11
What is the difference between the kill command and the killall command? Give an example of each.
Answer
kill terminates a process using its PID (Process ID) — e.g. kill -9 1234
killall (and pkill) terminate processes using the process name — e.g. killall firefox
The -9 flag sends a SIGKILL signal, which force-terminates the process immediately with no cleanup.
Question 12
Explain what umask 022 means. What permissions will a newly created directory have when this umask is active, and what will a new file have?
Answer
The umask value is subtracted from the system default permissions when creating new files/dirs.

Directory: Default is 777 → 777 − 022 = 755 (rwx r-x r-x) — owner has full control, group and others can read and execute.
File: Default is 666 → 666 − 022 = 644 (rw- r-- r--) — owner can read/write, group and others can only read. Files don't get execute by default.
