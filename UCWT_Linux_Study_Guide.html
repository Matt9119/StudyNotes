1a — NIX Characteristics
UNIX basics
Two distros covered
Ubuntu (Debian-based) and CentOS (RedHat-based). Run cat /etc/*release to identify which one.
Core OS functions
Runs programs, provides a UI, handles I/O and file system manipulation.
File identification
Each file type has a unique magic number — bytes at the start of a file that identify its type.
Key characteristics
Flexible, portable, case sensitive, multitasking, multi-user.

Hardware layer model

Kernel
Core of the OS. Creates/schedules/terminates processes, manages memory, filesystem, I/O, and error handling.
Shell
User-facing command interpreter. Passes commands to the kernel, handles I/O redirection, manages the environment.
4-step flow
User ↔ Shell ↔ Kernel ↔ Hardware
Exam trap

Linux is case sensitive everywhere — file names, commands, flags. Cat ≠ cat.

Permissions

Three permission types
r=4 Read   w=2 Write   x=1 Execute
Three user categories
u=User (owner)  |  g=Group  |  o=Others
Octal example
chmod 755 file → rwx r-x r-x
Symbolic example
chmod u=rw,g=r,o=r file
umask
Subtracted from defaults (dirs:777, files:666). umask 022 on dir → 755.

Special permissions

PermissionValueEffect
SUID4 / u+sExecutable runs as file owner
SGID2 / g+sExecutable runs as group owner
Sticky1 / o+tOnly file/dir owner can delete files
Set with 4th octal digit: chmod 4755 file — lowercase s/t = execute bit set; uppercase S/T = not set.
Pro tip

SUID on /bin/passwd is how a regular user can change their own password — it briefly runs as root.

1c — Boot Process
4 boot stages
1
Firmware (BIOS/UEFI)
Runs POST → discovers and loads bootloader into memory.
2
Boot Loader
Loads the kernel into working memory. GRUB is most common. May let user choose kernel.
3
Kernel
Starts /sbin/init and loads processes needed for the OS.
4
User Space
User can now interact with the system.

Init versions compared

Init typeKey traits
SysVSequential scripts in /etc/rcX.d/. Run levels 0–6. Not multi-threaded.
UpstartEvent-based service management. Uses initctl.
SystemdUses targets instead of run levels. Fastest. Most modern distros use this.
Key takeaway

Boot config files live in /boot/. Systemd default target is linked from /etc/systemd/system/default.target.

2a — Account Administration
Account types
Root (Superuser)
UID = 0. Unrestricted access to everything on the system.
System accounts
Run OS components (e.g., mail, sshd). Modifying privileges can break things.
User accounts
Interactive access for humans. Low-privilege and limited access.

Key account files

/etc/passwd
Format: username:pw:UID:GID:comment:home:shell. Defines primary group. Holds most account info.
/etc/shadow
Holds hashed passwords + aging policy (min/max age, expiration, warning period).
/etc/group
Additional group info per account.
/etc/gshadow
Secure group account info. Group admin is defined here.
/etc/sudoers
Edited with visudo. Default: root ALL=(ALL) ALL. Use %groupname to assign sudo to a group.
/etc/security
Security configs that restrict/limit system access.

Login info files (binary — use special commands)

These login files are binary — you can't read them with cat. You must use the dedicated commands (last, lastb, etc.).

2b–2f — Processes
Process fundamentals
PID
Every running process has a unique Process ID. init is always PID 1.
PPID
Parent Process ID — who spawned this process.
/proc/
Pseudo-filesystem containing info on all currently running processes.

System calls

SyscallPurpose
exec()Load a program into memory and execute it
fork()Spawn a child process (copy of running process)
exit()A process terminates itself
kill()A process terminates another; typically kill -9 <PID>

Process states

Zombie
Process has ended but still has an entry in the process table. Shows as defunct.
Orphan
Child process whose parent terminated without waiting for it. Still running.
Reaping
Parent reads child exit info via wait(). Until then, child shows as defunct.
Exam trap

kill and killall differ: kill uses PID; killall and pkill use the process name.

Suspicious process detection

  • Check for unusual open ports: sudo ss -tupl
  • Dump process info: sudo ls -al /proc/<pid>
  • Copy the binary and hash it to compare against known-good values.

Both netstat and ss must be run with superuser privileges to see associated PIDs/processes.

Networking tools

Log into a remote machine securely. Specify a command to run it remotely without an interactive shell.
scp
Securely copy files between systems.

Create/modify firewall rules. Chains: INPUT, OUTPUT, FORWARD. Targets: ACCEPT, DROP, REJECT (terminating) or LOG, RETURN, GOTO (non-terminating).
wget
Download files or web pages.
curl
Transfer data to or from a server.

Key takeaway

All logs live under /var/log/. The rsyslog daemon routes them to the right file. Logs rotate once a file hits capacity.

3b — Crontab
Scheduling tasks
Crontab format
Minute · Hour · Day-of-Month · Month · Day-of-Week · Command
crontab -l
List scheduled tasks for current user.
crontab -e
Edit crontab for current user.
crontab -r
Remove crontab for current user.
Access control
/etc/cron.allow and /etc/cron.deny limit who can use cron.
Pro tip

A scheduled task is called a cron job. Use an asterisk * in a field to mean "every" — e.g., * * * * * = every minute.

Scripting
Shebang line
First line of every script: #!/bin/bash. Tells the OS which interpreter to use.
Benefits
Reduces mistakes, enables reuse, saves time, adaptable to different environments.
Important note
Scripts do not change weapon systems under strict version controls.
