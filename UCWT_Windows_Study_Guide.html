1a — Windows system basics
Boot process
1
Preboot (BIOS/UEFI)
UEFI runs POST → reads MBR → finds and runs bootmgr
2
Windows Boot Manager
Reads Boot Config Database → loads Winload.exe
3
Boot Loader
Winload.exe loads drivers → Boot Manager + Loader load kernel into memory
4
Kernel
Loads registry into memory → launches SMSS.exe (Session Manager)

Key components

Processor
Executes instructions. Runs in User Mode (restricted/private) or Kernel Mode (privileged/shared). Architectures: x86 (32-bit) or x64.
API
Toolbox giving apps access to OS features — security, unified interface, compatibility.
Kernel
Bridge between user and hardware. First program loaded after bootloader. Windows uses a Hybrid Kernel.
Drivers
Enable OS-hardware communication. Software drivers always run in Kernel Mode.
DLLs
Shared libraries used by multiple programs simultaneously. Run in the calling process's memory space.

Processes

Services
Programs that run in the background. May contain multiple threads.
Session (SMSS)
First user-mode process started by kernel → starts winlogon.exe for interactive logons.
System Idle
Keeps processor occupied to prevent freezing. CPU % in Task Manager shows inverse of available power.
Exam trap

System Idle CPU % shows how much processing power is available, not how much is being used.

Common CMD commands

Lists directory contents. Hidden, read-only, and system files are NOT shown by default.

Prints to screen. Use > to overwrite, >> to append.
tasklist
View running processes. /svc shows services in each process.
taskkill
Kill a process by PID or image name.
sc
Manage services. Sub-commands: query, start, stop, qc, config, getkeyname.
net
Network resources, users, services. net start uses display name; sc start uses service key name.
find
Search strings in files. No regex.
findstr
Search strings in files. Supports regex.
systeminfo
Shows OS config, service packs, hotfixes.
set
Assign/view env variables. Access with %var_name%.

2a — Windows Registry
Registry basics
  • Keys — like folders in a file system
  • Values — like files in a file system
  • Data — content determined by the value's type
Key takeaway

The registry is a central hierarchical database storing all system configuration. Access it with regedit.

Registry hives

HKEY_CLASSES_ROOT
HKCR Linked
App associations for opening files. → HKLM\SOFTWARE\Classes
HKEY_CURRENT_USER
HKCU Linked
Profile of current logged-on user. → HKU\SID
HKEY_LOCAL_MACHINE
HKLM Root
Hardware and software config info for the OS.
HKEY_USERS
HKU Root
All loaded user profiles.
HKEY_CURRENT_CONFIG
HKCC Linked
Hardware profile at startup. → HKLM\SYSTEM\CurrentControlSet\HardwareProfiles\Current
Pro tip

Root hives = HKLM and HKU. HKCR, HKCU, HKCC are linked hives — just shortcuts.

2b — File System
FAT vs NTFS
FeatureFATNTFS
EncryptionNoneEFS supported
PermissionsNoneUser/Group ACLs
Common useUSB/SD cardsWindows drives
FAT32 max file4 GBNo practical limit
Exam trap

Copying a file from NTFS to FAT removes its security permissions and attributes.

NTFS permissions — access rules

Local access
Only NTFS permissions apply.
Share access
Both NTFS and Share permissions apply — most restrictive wins.
Group membership
User permissions are cumulative with group permissions.
Precedence
Explicit Deny > Explicit Allow > Inherited Deny > Inherited Allow

Copy, move & inheritance

ActionPermissions result
Copy within partitionInherits target folder perms
Move within partitionKeeps original perms
Move across partitionsInherits target folder perms
Move NTFS → FATLoses all attributes & security

File timestamps

ActionWhat changes
Copying a fileNew file: creation + access updated; write unchanged
Moving a fileAccess time modified
Opening from GUIAccess time modified
Appending (>>)Access + write time modified

Important file locations

User home dirs
Vista–Win11: C:\Users | XP: C:\Documents and Settings
System32 / SysWow64
x86 32-bit → System32 | x64 32-bit → SysWow64 | x64 64-bit → System32
Program Files
x86: C:\Program Files | x64: both Program Files folders
Exam trap

x86 = 32-bit. SysWow64 holds 32-bit programs on a 64-bit system — the name is counterintuitive.

Replica of all directory objects for the entire forest.

String showing an object's exact location in AD.

Unique number used as a persistent identifier.

AD structure hierarchy

ConceptDescription
ForestsOne or more domain trees without a contiguous namespace
Domain TreesDomains sharing schema/config in a contiguous namespace
OUsLogically grouped objects within a domain

Key takeaway

Forest > Tree > Domain > OU. The Global Catalog is built automatically by AD DS replication.

Logs are stored at C:\Windows\System32\winevt\logs. Configure auditing via secpol.msc (Local Security Policy).

Scheduled tasks

  • Enable automation of routine tasks.

  • Works on both local and remote systems.
