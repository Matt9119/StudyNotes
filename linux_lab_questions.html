File System Hierarchy Exploration & Boot Process Review
Day 1 Obj 1 · 3 Easy

You are orienting yourself to the layout of wrkst01. Your task is to explore key directories in the Linux file system hierarchy, document what lives where, and investigate which services and modules are currently loaded by the kernel.

  • List the top-level contents of the root (/) filesystem and briefly note the purpose of at least five directories you see.
  • Display the total size of the /var directory and the top five largest subdirectories within it.
  • Display all kernel modules currently loaded on the system.
  • Identify which runlevel or systemd target the system booted into, and list all services that are currently active.
  • Locate and display the contents of the file that defines which filesystems are automatically mounted at boot.

02
User & Group Environment Enumeration
Day 2 Obj 3 · 4 · 6 Easy

Before making any changes to wrkst01, you need a clear picture of the current user and group environment — who exists on the system, what groups they belong to, and what their session and environment look like.

  • Display the current user's identity, including their UID, GID, and all supplementary groups they belong to.
  • List all groups defined on the system and identify any groups that have more than one member.
  • Display all currently defined environment variables for your session.
  • Show which shell you are currently running and display its full path on the filesystem.
  • Display the contents of the current user's shell configuration file (e.g. .bashrc or .bash_profile) and identify any custom aliases or environment variables defined there.

03
User Account Audit & Failed Logon Analysis
Day 2 & 3 Obj 2 · 6 Medium

During a system integrity check on wrkst01, you must identify all human user accounts and cross-reference them against authentication failure logs to detect any accounts that have experienced failed login attempts.

  • Extract and display only human user accounts (UID ≥ 1000) from /etc/passwd, printing one username per line.
  • Using the appropriate log file(s) in /var/log, list all failed authentication attempts on the system.
  • Determine whether any of the human users identified in step 1 appear in the failed logon records. Document your findings.
  • Display the most recent login record for each user on the system.
Note: On RedHat, failed logins are recorded in /var/log/secure.

04
Process Listing & System Integrity Snapshot
Day 2 & 5 Obj 2 · 6 Medium

You need to evaluate the running process state of srvr01 after SSHing in. Your goal is to produce a comprehensive snapshot that captures the OS type, logged-on users, system uptime, and active process tree — all from a single terminal session.

  • Display all currently logged-on users along with their login time, idle time, and the command they are running.
  • Show the system uptime and current load average.
  • Capture a full process listing showing every process on the system with its PID, PPID, owner, CPU%, MEM%, and full command path.
  • Display the process tree to visualize parent-child relationships between running processes.
  • Identify any processes running as root that are listening on a network port — list the PID, process name, and port number.

05
File System Permissions & Attribute Security Analysis
Day 1 & 2 Obj 1 · 3 · 5 Medium

Perform a targeted file system security review on wrkst01. You are looking for files with elevated permissions, suspicious attributes, and access control configurations that could indicate privilege escalation paths or tampering.

  • Find all SUID and SGID binaries on the system (files with special permission bits set) and save the results to suid_files.txt.
  • Create a test file named immutable_test.txt in /tmp, set it as immutable, then verify the attribute is applied.
  • Display the ACL entries for /etc/passwd and /etc/shadow and identify which entries grant access beyond standard permissions.
  • Display the full metadata (permissions, inode, timestamps) for /etc/sudoers.
Note: SUID/SGID searches across the full filesystem may take some time to complete.

06
File Discovery, REGEX Filtering & Word Frequency Count
Day 4 & 5 Obj 3 · 5 Medium

A file containing the word "license" in its filename has been placed somewhere on wrkst01. You must locate it, extract specific word patterns from its content, and count how many words match the criteria.

  • Search the entire filesystem for any file whose name contains the string license (case-insensitive). Document the full path of the file found.
  • Using REGEX, extract all words from the file that are between 4 and 6 characters in length (alphabetic characters only).
  • Count the total number of words that fall within that character range and display only the count.
  • Sort the extracted words alphabetically, remove duplicates, and redirect the unique word list to a file named license_words.txt.

07
Crontab Audit & Scheduled Job Analysis
Day 3 Obj 3 · 5 Medium

Scheduled tasks are a common persistence mechanism. On wrkst01, perform a full audit of all cron-based scheduled jobs across every user and system-level location to determine what is configured to run automatically.

  • Display the crontab entries for the currently logged-in user, then display the system-wide crontab file.
  • List the contents of all system cron directories (/etc/cron.d, /etc/cron.daily, /etc/cron.weekly, /etc/cron.monthly) and note any scripts or jobs defined there.
  • Determine whether any other user accounts on the system have crontab entries configured. List the usernames and their scheduled jobs.
  • Create a new crontab entry for the current user that would run a script called ~/health_check.sh every day at 6:00 AM, then verify the entry was saved correctly.

08
Bash Scripting — User Account Reporter
Day 4 Obj 3 · 5 Medium

Write a Bash script on wrkst01 that iterates over all human user accounts on the system and reports key information about each one — combining file parsing, conditional logic, and loops into a practical reporting tool.

  • Create a script named user_report.sh in your home directory.
  • Using a loop, read through /etc/passwd and process only accounts with a UID of 1000 or greater. For each, print the username and their assigned home directory.
  • For each user found, add a conditional check: if their home directory exists on disk, print [OK] next to their entry — if it does not, print [MISSING].
  • Make the script executable, run it, and redirect the full output to a file named user_report.txt.

09
Disk & Package Inventory on a Remote System
Day 5 Obj 4 · 7 Medium

SSH into srvr01 from wrkst01 and perform a storage and software inventory. Understanding what is installed and how storage is allocated is a key part of assessing an unfamiliar system.

  • Display all currently mounted filesystems along with their type, size, used space, available space, and mount point.
  • Identify the single directory under / consuming the most disk space and display its total size.
  • List all installed packages on srvr01 and redirect the full list to a file named pkg_inventory.txt. Display only the total count of installed packages.
  • Filter the package list to show only entries related to networking tools. Display the results.
  • Transfer pkg_inventory.txt back to the ~/Desktop directory on wrkst01 using SCP.

10
Tactical Survey — Remote System Situational Awareness
Day 2 · 3 · 5 Obj 4 · 6 · 7 Medium

SSH into srvr01 from wrkst01 and conduct a structured situational awareness survey. Answer the key questions an analyst needs when first accessing an unfamiliar system, and transfer your findings back to your workstation.

  • Identity: Display the OS, kernel version, hostname, and current system uptime.
  • Users: Show all currently logged-on users and the last 10 system-wide login events.
  • Processes: Capture a full process listing showing the PID, owner, and command for every running process.
  • Storage: Display disk usage for all mounted filesystems in human-readable format.
  • Redirect all of the above output into a single file named survey_srvr01.txt on srvr01, then transfer it to ~/Desktop on wrkst01 using SCP.
